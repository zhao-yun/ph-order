# 创建订单（自动计算定价）接口文档

## 1. 接口概览

- **BaseUrl**：http://49.235.114.122:8080
- **接口名称**：创建订单（自动计算定价）
- **Method**：POST
- **Path**：`/Order/CreateOrderWithPricing`
- **Content-Type**：`application/json`
- **鉴权**：可选（当前不强制）
  - 前端需要传入 `userId`
  - 同时后端预留了“从 token 解析 userId 并覆盖前端 userId”的能力：当前实现为**假实现**（返回空字符串，不覆盖前端）

## 2. 服务类型枚举（前端传 type）

`type` 为 int64，取值范围 1-4：

| type | 服务类型 | 说明 |
|------|----------|------|
| 1 | Boarding（寄养） | 按天计费（FromDate~ToDate 含首尾） |
| 2 | Daycare（日托） | 按天计费（FromDate~ToDate 含首尾） |
| 3 | Walking（遛狗） | 当前按“每天一次/每次”计费（FromDate~ToDate 含首尾） |
| 4 | DropIn（上门） | 当前按“每天一次/每次”计费（FromDate~ToDate 含首尾） |

后端枚举定义见：[order_service_type.go](file:///Users/bytedance/workplace/go/src/demo/model/order_service_type.go)

## 3. 定价表（当前写死，后续按 sitterId 查询）

当前定价数据由后端写死返回（后续你可替换为从 DB/配置中心按 `sitterId` 查询）：

- 查询入口（预留方法）：[GetSitterPricingBySitterID](file:///Users/bytedance/workplace/go/src/demo/service/pricing_provider.go)

| PetList[].PetID | string | 是 | `"p1"` | 宠物ID（DB not null）。 |

| NeedPickup | bool | 否 | `true` | 是否需要接送。若 true，TotalPrice 额外 + 接送费（一次性）。 |
| DurationMinutes | int | 否 | `60` | 仅在 type=3(Walking) 生效。允许 30/60；若为 60，则“第一只狗”基础价使用 60分钟服务价覆盖默认基础价。 |

## 5. 返回定义

### 5.1 成功响应

统一返回结构（OpenAPI）：

```json
{
  "code": 0,
  "msg": "success",
  "data": { "..." : "..." }
}
```

`data` 为创建后的订单对象（包含 `PetList`），字段定义见：[Order](file:///Users/bytedance/workplace/go/src/demo/model/order.go)

关键字段说明：

| 字段 | 说明 |
|------|------|
| data.OwnerID | 最终写入订单的用户ID（token 解析到则覆盖，否则使用 userId） |
| data.Type | 服务类型（1-4） |
| data.SubTotalPrice | 宠物费用合计（单位：元，float64） |
| data.TotalPrice | 订单总价（单位：元，float64 = SubTotalPrice + 增值项） |
| data.PetList[].PetPrice | 该宠物在整个订单周期内的费用合计（单位：元，float64） |

### 5.2 失败响应（常见）

```json
{
  "code": 500,
  "msg": "xxx error message",
  "data": null
}
```

> 注：当前工程中 OpenApiErrorResponse 里 `code` 实际取决于实现；接口会设置 HTTP Status（400/500）。

常见错误（HTTP 400）：

| 场景 | 返回 msg（示例） |
|------|------------------|
| userId 为空 | `userId is required` |
| type 非法（不在1-4） | `invalid Type: 99` |
| 日期缺失/格式错误 | `FromDate/ToDate is required` / common.Date 解析错误 |
| ToDate < FromDate | `ToDate must be >= FromDate` |
| PetList 为空 | `PetList is empty` |
| PetType 不支持 | `unsupported PetType: xxx` |
| Walking 的 DurationMinutes 非 30/60 | `invalid DurationMinutes: 45` |

## 6. 计价规则（实现细则）

计价核心实现见：[CalculateOrderPricing](file:///Users/bytedance/workplace/go/src/demo/service/order_pricing.go)

### 6.1 基本口径

- Boarding/Daycare：按天计费
- Walking/DropIn：当前按“每次”计费，但实现上仍用日期范围推导“次数 = 天数”（FromDate~ToDate 含首尾）

天数/次数计算：`units = (ToDate - FromDate) + 1`

### 6.2 宠物计费规则

#### 狗（Dog）

- 第 1 只狗：基础价默认 `DogBaseCents`
  - 可能被以下规则覆盖（优先级见 6.3）
- 第 2 只及以后：每只按 `AdditionalDogCents`

#### 猫（Cat）

- 第 1 只猫：按 `CatExclusiveCents`（猫专属价，即猫的基础价）
- 第 2 只及以后：每只按 `AdditionalCatCents`（附加猫费）

示例：3只猫寄养1天 = `猫专属价 4500 + 附加猫费 2000*2 = 8500`

### 6.3 “第 1 只狗”的基础价覆盖优先级

对每个计费单元（每天/每次），第1只狗基础价按如下优先级选择：

1. **Walking 且 DurationMinutes=60**：使用 `Walking60MinCents`
2. **节假日**：使用 `HolidayCents`
3. **Boarding 且长期(>=7天)**：使用 `LongTermCents`
4. **幼犬(IsPuppy=true)**：使用 `PuppyCents`
5. 默认：使用 `DogBaseCents`

### 6.4 节假日判定

当前内置 **2026 年中国法定节假日放假区间**（按日期命中）：

- 元旦：2026-01-01 ~ 2026-01-03
- 春节：2026-02-15 ~ 2026-02-23
- 清明：2026-04-04 ~ 2026-04-06
- 劳动节：2026-05-01 ~ 2026-05-05
- 端午：2026-06-19 ~ 2026-06-21
- 中秋：2026-09-25 ~ 2026-09-27
- 国庆：2026-10-01 ~ 2026-10-07

实现见：`isHolidayCN(date)`（[order_pricing.go](file:///Users/bytedance/workplace/go/src/demo/service/order_pricing.go)）

> 若你需要支持跨年/多地区/动态配置，后续建议将节假日表做成配置或DB表。

### 6.5 增值项（一次性加价）

- `NeedGrooming=true`：总价 + `GroomingCents`
- `NeedPickup=true`：总价 + `PickupCents`

> 注意：增值项目前不按天累加，按一次性累加（符合你文档示例：3天寄养 + 洗澡美容 = 5000*3 + 1500）。

## 7. 写库行为（数据落地）

接口会写入两张表（由 DAL 事务保证一致性）：

- `orders`：见 [Order](file:///Users/bytedance/workplace/go/src/demo/model/order.go)
  - `OwnerID = 最终 userId`
  - `SitterID = 请求 SitterID`
  - `Type = 请求 type`
  - `FromDate/ToDate = 请求日期`
  - `SubTotalPrice/TotalPrice` 写入计算结果（单位：元）
  - `State = OrderInitialized(0)`
- `order_pet`：见 [OrderPet](file:///Users/bytedance/workplace/go/src/demo/model/order_pet.go)
  - 每只宠物写一条，`PetPrice` 为该宠物在整个周期内累积费用（单位：元）

DAL 见：[CreateOrder](file:///Users/bytedance/workplace/go/src/demo/dal/order_dal.go#L132-L176)

  - 当前实现返回空字符串（不覆盖）
