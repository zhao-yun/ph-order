# 遛狗轨迹模块接口文档

## 概述

遛狗轨迹模块用于记录和查询遛狗订单的实际行走轨迹数据，仅适用于遛狗订单（Type=3）。

**重要说明：** 
- 一个子订单对应一个轨迹记录
- 支持批量追加轨迹点
- 子订单中有专门字段存储轨迹缩略图URL
- 查询时自动计算轨迹时长和实际距离
- 支持实时显示Sitter位置

## 数据模型

### LatLng
经纬度坐标点，带时间戳
```typescript
type LatLng = {
  lat: number;       // 纬度
  lng: number;       // 经度
  timestamp?: number; // Unix时间戳（毫秒），用于计算轨迹时长
};
```

### WalkDetail（轨迹详情）
```typescript
interface WalkDetail {
  id: string;
  orderId: number;                     // 主订单ID
  subOrderId: number;                  // 子订单ID
  path: LatLng[];                      // 完整路径点数组
  durationSeconds: number;             // 轨迹时长（秒），从第一个点到最后一个点的时间间隔
  distanceMeters: number;              // 轨迹实际距离（米），非直线距离，根据路径点计算
  walkThumbnailUrl: string;            // 轨迹缩略图URL
  createdAt?: string;                  // 创建时间
}
```

### SitterLocation（Sitter位置）
```typescript
interface SitterLocation {
  id: number;         // 位置记录ID
  sitterId: string;   // Sitter ID
  orderId: number;    // 主订单ID
  subOrderId: number; // 子订单ID
  lat: number;        // 纬度
  lng: number;        // 经度
  timestamp: number;  // 时间戳（毫秒）
  createdAt: string;  // 创建时间
}
```

## 接口列表

### 1. 追加轨迹点

**接口地址：** `POST /WalkRecord/AppendWalkPath`

**请求参数：**
```json
{
  "SubOrderID": 456,
  "Path": [
    {"lat": 40.7128, "lng": -74.0060, "timestamp": 1710936000000},
    {"lat": 40.7138, "lng": -74.0070, "timestamp": 1710936060000},
    {"lat": 40.7148, "lng": -74.0080, "timestamp": 1710936120000}
  ]
}
```

**参数说明：**
- `SubOrderID`: 子订单ID（必填）
- `Path`: 要追加的路径点数组（必填）
  - `lat`: 纬度（必填）
  - `lng`: 经度（必填）
  - `timestamp`: Unix时间戳（毫秒），建议提供以便准确计算时长

**响应示例：**
```json
{
  "Code": 0,
  "Data": {
    "id": 1,
    "orderId": 123,
    "subOrderId": 456,
    "path": [
      {"lat": 40.7128, "lng": -74.0060, "timestamp": 1710936000000},
      {"lat": 40.7138, "lng": -74.0070, "timestamp": 1710936060000},
      {"lat": 40.7148, "lng": -74.0080, "timestamp": 1710936120000}
    ],
    "createdAt": "2026-03-20 10:00:00"
  },
  "Message": "success"
}
```

**注意事项：**
- 如果该子订单已有轨迹记录，新的路径点会追加到现有路径后面
- 如果没有轨迹记录，则创建新记录
- 返回的是合并后的完整路径
- **自动更新实时位置**：该接口会自动提取传入的 `Path` 中时间戳最新的点，作为 Sitter 的当前实时位置并更新到位置表中，前端无需再单独调用 `SitterLocation/Update` 接口。

### 2. 根据子订单ID查询轨迹

**接口地址：** `GET /WalkRecord/GetWalkRecordBySubOrderId`

**请求参数：**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| SubOrderID | string | 是 | 子订单ID |

**响应示例（有轨迹）：**
```json
{
  "Code": 0,
  "Data": {
    "id": "1",
    "orderId": 123,
    "subOrderId": 456,
    "path": [
      {"lat": 40.7128, "lng": -74.0060, "timestamp": 1710936000000},
      {"lat": 40.7138, "lng": -74.0070, "timestamp": 1710936060000},
      {"lat": 40.7148, "lng": -74.0080, "timestamp": 1710936120000}
    ],
    "durationSeconds": 120,
    "distanceMeters": 250.5,
    "walkThumbnailUrl": "https://example.com/thumbnail.jpg",
    "createdAt": "2026-03-20 10:00:00"
  },
  "Message": "success"
}
```

**响应示例（无轨迹）：**
```json
{
  "Code": 0,
  "Data": {
    "walkThumbnailUrl": "https://example.com/thumbnail.jpg"
  },
  "Message": "success"
}
```

**字段说明：**
- `durationSeconds`: 轨迹时长，单位秒。从第一个轨迹点的timestamp到最后一个轨迹点的timestamp的时间差
- `distanceMeters`: 轨迹实际距离，单位米。使用Haversine公式计算路径点之间的累计距离，不是直线距离
- `walkThumbnailUrl`: 子订单的轨迹缩略图URL，从子订单表获取

**注意事项：**
- 如果该子订单没有轨迹记录，仍会返回 `walkThumbnailUrl` 字段（如果有设置）
- 如果路径点少于2个，`durationSeconds` 和 `distanceMeters` 返回0

**接口地址：** `POST /SubOrder/UpdateWalkThumbnail`

**请求参数：**
```json
{
  "SubOrderID": 456,
  "Url": "https://example.com/thumbnail.jpg"
}
```

**参数说明：**
- `SubOrderID`: 子订单ID（必填）
- `Url`: 轨迹缩略图URL（必填）

**响应示例：**
```json
{
  "Code": 0,
  "Data": {
    "id": 456,
    "orderId": 123,
    "date": "2026-03-20",
    "state": 2,
    "WalkThumbnailUrl": "https://example.com/thumbnail.jpg"
  },
  "Message": "success"
}
```

**注意事项：**
- 查询子订单列表或详情时，`WalkThumbnailUrl` 字段会一起返回
- 查询轨迹时，也会同时返回该字段

## 计算逻辑

### 轨迹距离计算
使用 **Haversine公式** 计算地球表面两点之间的真实距离，然后累加所有相邻点的距离得到总距离。这不是直线距离，而是实际行走的路径距离。

### 轨迹时长计算
从所有路径点中找到最小的timestamp（最早时间）和最大的timestamp（最晚时间），两者的差值即为轨迹时长。

## 使用说明

1. **仅遛狗订单可用**：只有订单类型为遛狗（Type=3）的订单才能创建轨迹记录
2. **一对一关系**：一个子订单对应一个轨迹记录
3. **追加模式**：每次调用 `AppendWalkPath` 都会将新的路径点追加到现有路径后面
4. **路径数据**：路径数据以JSON格式存储在数据库中，包含完整的经纬度坐标点数组和时间戳
5. **轨迹缩略图**：通过 `UpdateWalkThumbnail` 接口设置子订单的轨迹缩略图URL，查询子订单和轨迹时都会返回该字段
6. **自动计算**：查询轨迹时自动计算并返回 `durationSeconds`（时长）和 `distanceMeters`（实际距离）
7. **实时位置**：当调用 `AppendWalkPath` 追加轨迹点时，后端会自动提取最新点更新为 Sitter 实时位置。
8. **位置查询**：通过 `SitterLocation/GetLatest` 接口获取Sitter最新位置，可用于实时显示
